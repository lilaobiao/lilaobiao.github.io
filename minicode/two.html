# 起步

因为通过开发工具创建的小程序，就是一个标准的小程序代码结构，所以我们先创建一个小程序，再讲小程序的代码结构，这样跟着代码查看更有助于理解。

# 创建一个小程序

打开开发工具，使用你注册小程序账号时的微信扫描二维码登录。登陆后选择 小程序项目，点击右下角的加号新增项目，即可打开新增界面。项目目录即项目存放的目录，可自行选择，但必须得是个空文件夹才行，可先自行创建一个，AppID填写我们注册账号的appid，项目名称随便填写，支持中文，最后选择 建立普通快速启动模板。如下所示：

点击确定，这时，我们的第一小程序就算创建好了，开发工具界面如下：
右边是小程序代码，而左边则是预览效果。可以看到，这时代码已经成功获取了我们的用户名和头像。

# 小程序代码结构

我们的小程序代码结构如图所示：

上图也是一个小程序最基本的代码结构。可以看到，小程序里面并没有html和css文件，这就是小程序和普通web页面的不同之处。在小程序里，wxml 为页面文件，和 html 写法一样，但却有一套自己的标签，像 html 里的 div、span、p这些都不能用，wxss 为样式文件，写法同 css 一样，支持样式选择器，id选择器和标签选择器，以及 :before 和 :after，其他的 :first-child，nth-child 等不受支持。

# 代码分析

pages 是一个文件夹，里面存放的是小程序的所有的页面。而每个页面又是一个单独的文件夹，由四个文件组成，分别是 .js、.json、.wxml 和 .wxss，四个文件的文件名必须相同才行，但可以和文件名不同。所有的文件，都只在当前页面有效，包括js里面定义的变量和样式文件。

utils 里面存放的是工具类js，可以在其他js里面引入使用。

app.js 是小程序的入口api，里面创建了一个 App 实例，且每个小程序只能创建一个 App 实例，在小程序的所有页面都能获取到该实例，可用于存储全局数据。

//app.js
App({
  onLaunch: function () { // 小程序首次加载时调用
    ...
  },
  globalData: { // 全局参数，贯穿整个小程序
    userInfo: null
  }
})

app.json 是小程序的配置文件，里面存放小程序的页面信息和通用配置。

{
  "pages":[//没有在这里指定的页面，是无法访问的
    "pages/index/index",//每一项都是一个页面路径，指向后缀为.wxml的文件
    "pages/logs/logs"
  ],
  "window":{//小程序全局window窗口配置，可以在页面的json文件里面重写覆盖
    "backgroundTextStyle":"light", //下拉 loading 的样式，仅支持 dark / light
    "navigationBarBackgroundColor": "#fff", //导航栏背景颜色，如 #000000
    "navigationBarTitleText": "WeChat", //导航栏标题文字内容
    "navigationBarTextStyle":"black" //导航栏标题颜色，仅支持 black / white
  }
}

更多配置参考 全局配置

project.config.json 是开发工具配置文件，一般我们可以不用理会。它的作用是保存我们当前小程序的关于开发工具的相关配置，包括我们自己添加的编译模式等信息，以保证我们以后在别的电脑上的开发工具里打开时，和在自己的电脑上看到的一样。

app.wxss 是小程序的全局样式文件，相当于一般项目的 main.css。
